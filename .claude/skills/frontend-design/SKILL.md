このスキルは、一般的な「AI臭い」デザインを避けた、独創的で本格的なフロントエンドインターフェースの作成をガイドします。美的細部とクリエイティブな選択に特別な注意を払った、実際に動作するコードを実装します。

ユーザーは、構築するコンポーネント、ページ、アプリケーション、またはインターフェースというフロントエンド要件を提供します。目的、対象ユーザー、技術的制約に関するコンテキストも含まれる場合があります。

## プロジェクト固有の技術制約

このプロジェクトでは以下の技術スタックを使用します:

- **フレームワーク**: Next.js 16.0.4 (App Router)
- **言語**: TypeScript (strict モード)
- **スタイリング**: Tailwind CSS v4 (PostCSS) - **CSS変数ではなくTailwindユーティリティクラスを優先**
- **UIコンポーネント**: shadcn/ui (New Yorkスタイル) - **既存のUIライブラリを優先使用**
- **画像**: Next.js Image コンポーネント必須
- **レスポンシブ**: モバイルファースト設計 (768px以上がPC判定)

### 必須要件

- **画像最適化**: `next/image`コンポーネントを使用し、適切なsizes・priority・loading属性を設定
- **SEO対策**: JSON-LD構造化データ、パンくずリスト、適切なメタタグを含める
- **パフォーマンス**: 最初の3要素以外は遅延読み込み(lazy loading)を実装

### デザインの方向性

- **視認性重視**: プロフィール画像・スペック情報が常に主役
- **清潔感**: 過度に派手なデザインを避け、情報の見やすさを優先
- **カードデザイン**: 3:4のアスペクト比を基準とする

## デザイン思考

コーディングの前に、コンテキストを理解し、プロジェクトに適した美的方向性にコミットします:

- **目的**: このインターフェースは何の問題を解決するか? 誰が使用するか?
- **トーン**: プロジェクトの性質に合わせた適切な方向性を選択: 極限のミニマル、マキシマリストのカオス、レトロフューチャー、有機的/自然的、高級/洗練、遊び心/おもちゃ風、エディトリアル/雑誌風、ブルータリスト/生々しい、アールデコ/幾何学的、柔らか/パステル、工業的/実用的など。
- **制約**: 技術要件(上記の技術スタック、パフォーマンス、アクセシビリティ)
- **差別化**: 何が忘れられないものにするか? ユーザーが覚えている一つのことは何か?

**重要**: 明確な概念的方向性を選択し、精度を持って実行します。大胆なマキシマリズムと洗練されたミニマリズムの両方が機能します - 鍵は強度ではなく、意図性です。

その後、以下のような動作するコード(TypeScript/React/Next.js)を実装します:

- 本番環境品質で機能的
- 視覚的に印象的で記憶に残る
- 明確な美的観点を持って統一的
- すべての細部まで細心の注意を払って洗練
- shadcn/uiコンポーネントとの統合

## フロントエンド美学ガイドライン

以下に焦点を当てます:

### タイポグラフィ

- 美しく、ユニークで、興味深いフォントを選択します
- ArialやInterのような一般的なフォントは避け、独特な選択を優先
- 予期しない、個性的なフォント選択
- 独特なディスプレイフォントと洗練された本文フォントをペアにします
- Google Fontsなどから適切なフォントを選択し、`next/font`で最適化

### カラー&テーマ

- 統一的な美学にコミットします
- **Tailwind CSSのユーティリティクラスを使用** (例: `bg-blue-100`, `text-gray-800`)
- 鋭いアクセントを持つ支配的な色は、臆病で均等に分散されたパレットよりも優れています

### モーション

- エフェクトとマイクロインタラクションにアニメーションを使用
- **Tailwindのtransitionユーティリティを優先** (例: `transition-shadow duration-300`)
- 高インパクトの瞬間に焦点: 時差表示(animation-delay)を持つ調整されたページロードが喜びを生み出します
- スクロールトリガーと驚きを与えるホバー状態を使用

### 空間構成

- 予期しないレイアウト
- 非対称性、重なり、斜めの流れ
- グリッドを破る要素
- 寛大なネガティブスペース、または制御された密度
- **レスポンシブ対応**: モバイル(〜767px)とPC(768px〜)で最適な表示

### 背景&ビジュアル詳細

- 単色にデフォルトするのではなく、雰囲気と深みを作成
- 全体的な美学に合致する文脈的エフェクトとテクスチャを追加
- ノイズテクスチャ、幾何学パターン、レイヤー透明度、劇的な影、装飾的な境界線などのクリエイティブな形式を適用

## 禁止事項

以下のような一般的なAI生成の美学は絶対に使用しないでください:

- 使い古されたフォントファミリー(Inter、Roboto、Arial、システムフォント)
- 陳腐なカラースキーム(特に白背景の紫グラデーション)
- 予測可能なレイアウトとコンポーネントパターン
- コンテキスト固有の特性を欠く型通りのデザイン

クリエイティブに解釈し、コンテキストのために本当にデザインされたと感じられる予期しない選択をします。同じデザインは存在しないはずです。ライトテーマとダークテーマ、異なるフォント、異なる美学の間で変化させます。世代を超えて共通の選択(例えばSpace Grotesk)に収束しないでください。

## 実装の複雑さ

実装の複雑さを美的ビジョンに合わせます:

- **マキシマリストデザイン**: 広範なアニメーションとエフェクトを持つ精巧なコード
- **ミニマリスト/洗練されたデザイン**: 抑制、精度、間隔・タイポグラフィ・微妙な詳細への注意深い配慮
- エレガンスはビジョンをうまく実行することから生まれます

覚えておいてください: Claudeは並外れたクリエイティブな作業が可能です。遠慮せず、枠にとらわれない思考と独創的なビジョンへの完全なコミットメントで、本当に創造できるものを示してください。
